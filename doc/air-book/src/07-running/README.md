# Running the AIR Tool

This section walks through the complete AIR Tool workflow from preparing your input files to interpreting your results. Each step of the analysis has its own subchapter.

## Workflow Overview

1. **[Input Data Files](input-files.md)** — Prepare your data file and knowledge file to meet AIR Tool requirements
2. **[Step 1: Causal Graph](step1-causal-graph.md)** — Upload your files and build the causal graph
3. **[Step 2: Select Variables](step2-variables.md)** — Identify your treatment and outcome variables
4. **[Step 3: Estimating Causal Effect](step3-estimation.md)** — Provide or generate a model ATE for comparison
5. **[Interpreting Results](results.md)** — Read and act on the AIR health report
6. **[Usage Scenarios](scenarios.md)** — Use the scenario definition worksheet to plan and document your analysis
7. **[Troubleshooting](troubleshooting.md)** — Retrieve logs and resolve common problems

## Before You Begin

Before beginning analysis with the tool, it is useful to carefully consider the causal scenario that the classifier model is used to solve. Understanding the scenario variables to be analyzed and confirming completeness of the data will contribute to gaining meaningful insight from the AIR results.

More detailed information about preparing the scenario and data for analysis can be found in the [Scenario Definition Worksheet](scenarios.md) and [Dataset Guidelines](../06-data-guidelines/README.md).
